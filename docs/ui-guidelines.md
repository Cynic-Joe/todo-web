# UI 设计规范

后续只要涉及页面、组件、排版、视觉或文案改动，先遵循这份文档。

## 硬约束

- 不要国风、古风、旧报刊、厚重复古纸张这类强风格视觉。
- 不要英文导航、英文副标题、英文状态文案；界面默认中文，只有 `GitHub`、`Gist` 这类专有名词可以保留英文。
- 不要恢复大 Hero、大摘要区、展示型说明卡、概念卡或侧栏总结卡。
- 所有中文保持正常 UTF-8，禁止乱码。

## 信息架构

- 主工作区保持单列，优先展示当前正在操作的内容。
- 当前主功能固定为：`待办`、`搁置`、`创意`、`完成`、`记账`。
- 导航只用中文章节切换，不加英文副标题。
- 各板块头部只保留主标题和计数，不额外加辅助说明文字。
- `创意` 为独立列表，和 `待办` 分开存储与展示。
- `创意` 页只提供三类操作：新增创意、转待办、删除；不支持直接完成。
- 来源于 `创意` 的条目转入 `待办` 后，需要在后续进入 `完成` 页时保留 `创意` 来源标记。

## 默认方向

- 整体保持现代、简洁、轻量，像干净的记录工具。
- 各区块尽量共用同一套容器、按钮、输入框和空状态语言，不为每个模块单独发明一套视觉。
- 通过状态、时间、金额和轻微色彩区分内容，不靠重装饰分区。
- 文案只回答三件事：这里是什么、现在有没有内容、下一步能做什么。
- 交互保持直接，常用操作尽量在当前语境内完成。
